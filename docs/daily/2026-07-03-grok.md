## Hot Discussions

- **Harness differences dominate outcomes more than underlying models**: Parallel experiments (Fable/Claude Code vs GPT-5.5 in Codex vs same model in Grok Build vs Cursor-agent) showed dramatically different behavior and results despite identical prompts/models. Discussion emphasized that scaffolding (subagent dispatch, context isolation, sandboxing, orchestration primitives) is the primary lever; "harness matters more than we think."

- **Traces as the operating system for agent reliability and improvement**: Consensus that passive logs are insufficient. Full trajectories (tool calls, state, errors) feed LangSmith Engine, Harbor evals, rubric graders, and custom loops. Key quote: "you don't know what your agents will do until you actually run them." Evals must shift to replayable, environment-aware trajectory evaluation rather than final-answer scoring.

- **From ephemeral context to agent-maintained structured knowledge**: Strong agreement that RAG-per-query rediscovers knowledge every time while wiki-style memory compounds. Agents "love markdown files." Debate centers on maintenance loops (automation vs human curation), staleness detection, provenance, integration with git/Obsidian/MCP, and using the wiki as primary reasoning substrate. Ties directly into self-improving loops and Deep Agents patterns.
